深入ES6(一) let与const

##let命令

###基本用法
ES6新增了let命令，用来声明变量。它的用法类似于var，但是所声明的变量，只在let命令所在的代码块内有效。

用let做循环处理就会比较方便,不会使i作用于外面。

用let时,因为变量是作用于遍历块内的,所以取过来的值是即时的,而var声明的是全局变量,调用时全局变量已经走完循环变成了10

let 不能创建名字一样的变量,如果变量名一样会报错

let和var的报错提示也是不一样的

只要有let的存在,那么这个变量只存在于该作用域内

####暂时性死区:

只要一进入当前作用域，所要使用的变量就已经存在了，但是不可获取，只有等到声明变量的那一行代码出现，才可以获取和使用该变量。

let不允许重复声明变量

下面代码中外层代码块不受内层代码块影响

##const命令

基本用法:

const声明一个只读的常量。一旦声明，常量的值就不能改变。

一旦常量被更改,将会报错

声明const时必须有初始值,因为const不能被修改,所以当声明没有初始值的const会报错

同let一样,const也存在暂时性死区

如果一个变量被声明过,const再来声明也会报错

const的实际目的并不只是使该变量不可改动,实际意义在于const指向的该变量内存地址不可改动

###申明变量的六种方式:

1、var
2、function
3、let
4、const
5、import
6、class

不要再说只有一个var了 ✧(≖ ◡ ≖✿)嘿嘿

如果有时间后面会讲讲import和class

细讲encodeURI和encodeURIComponent以及escape的区别与应用

首先，我们都知道这三个东西都是用来编码的

先来说encodeURI()和encodeURIComponent()

这两个是在转换url时候用来编码解码用的。

有编码就会有解码，解码就是decodeURI()和decodeURIComponent()

他们的用法很简单，在参数中带入要转码的文字就可实现目的

而encodeURI()和encodeURIComponent()的区别其实并不大
主要区别在于：


encodeURI不编码字符有82个：!，#，$，&，’，(，)，，+，,，-，.，/，:，;，=，?，@，，~，0-9，a-z，A-Z
encodeURIComponent不编码字符有71个：!， ‘，(，)，，-，.，，~，0-9，a-z，A-Z

而escape，相比于上面那两个，就有所不同了

escape()是编码，unescape()是解码；

escape 方法:

对 String 对象编码以便它们能在所有计算机上可读，

escape(charString)

必选项 charstring 参数是要编码的任意 String 对象或文字。

说明 :
escape 方法返回一个包含了 charstring 内容的字符串值（ Unicode 格式）。所有空格、标点、重音符号以及其他非 ASCII 字符都用 %xx 编码代替，
其中 xx 等于表示该字符的十六进制数。例如，空格返回的是 “%20” 。
字符值大于 255 的以 %uxxxx 格式存储。
escape不编码字符有69个：*，+，-，.，/，@，_，0-9，a-z，A-Z

注意 escape 方法不能够用来对统一资源标示码 (URI) 进行编码。对其编码应使用 encodeURI 和encodeURIComponent 方法。
